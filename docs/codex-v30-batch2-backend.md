## 一、POST /behavior 返回值增加 attrTempTotal（修改 server/routes/behavior.js）

在 `POST /` 路由中，行为记录和道具插入成功后、返回 `res.json` 之前，查询该属性维度的当前累计临时值：

在约第 195 行（`const checkinResult = doCheckin(...)` 之后）插入：

```js
// 查询该属性维度当前累计未使用的临时值总和
const attrField = CATEGORY_TO_ATTR[category];
const attrTempRow = db.prepare(
  `SELECT COALESCE(SUM(temp_value), 0) as total
   FROM items WHERE user_id = ? AND attribute_type = ? AND status = 'unused'`
).get(req.user.id, attrField);
const attrTempTotal = Math.round((attrTempRow?.total || 0) * 10) / 10;
```

然后在 `res.json` 的返回对象中增加 `attrTempTotal` 字段：

```js
res.json({
  behavior: { id: behaviorId, category, sub_type, quality },
  item: { id: itemId, name: item.name, quality: item.quality, attribute_type: item.attribute_type, temp_value: item.temp_value },
  cultivationStatus: cultivation,
  checkinResult,
  attrTempTotal,  // 新增：该属性当前累计临时值
});
```

## 三、generateItem 返回值增加描述（修改 server/services/itemGen.js）

在文件顶部引入描述数据：

```js
const itemDescriptions = require('../data/item-descriptions.json');
```

修改 `generateItem` 函数，在返回对象中增加 `description` 字段：

```js
function generateItem(category, quality) {
  const attrType = CATEGORY_TO_ATTR[category];
  if (!attrType) return null;

  const names = itemNames[category] || itemNames['默认'];
  const name = names[Math.floor(Math.random() * names.length)];
  const tempValue = QUALITY_VALUES[quality];
  const description = itemDescriptions[name] || '';

  return { name, quality, attribute_type: attrType, temp_value: tempValue, description };
}
```

同时修改 `POST /behavior` 的返回值，在 item 对象中增加 description：

```js
item: { id: itemId, name: item.name, quality: item.quality, attribute_type: item.attribute_type, temp_value: item.temp_value, description: item.description },
```

## 四、趋势图数据改为最右侧固定今天（修改 server/routes/character.js）

当前 `getRecentTrend` 函数（约第 35-78 行）生成最近 7 天的数据，从 6 天前到今天。这个逻辑本身已经是最右侧=今天。

确认当前实现：
```js
for (let i = 6; i >= 0; i--) {
  const d = new Date(now);
  d.setDate(d.getDate() - i);
  days.push(formatDate(d));
}
```

这已经是正确的（days[6] = 今天）。如果前端显示不对，问题在前端的 `buildTrendBars` 中 weekday 的显示。

**无需后端改动**，趋势图的日期顺序调整在前端指令中处理。
