# UI 设计规范

> **创建日期**：2026-04-14
> **来源**：V2.6 UI 审查 + 中国 app UI 范式调研
> **适用范围**：修仙日常 Web 游戏，手机端（375×667 视口，max-width 480px）
> **目标用户**：中国家庭用户（含中老年人）
> **技术基础**：单一 CSS 文件 `public/css/style.css`，CSS 自定义属性定义在 `:root`

---

## 一、色彩体系

深色主题，紫金配色（紫气东来、金丹大道）。品牌色使用面积控制在页面 10% 以内。

### 主色调

| 变量 | 色值 | 用途 |
|------|------|------|
| `--primary` | `#8b5cf6` | 紫，主色 |
| `--primary-hover` | `#7c3aed` | 主色 hover 态 |
| `--primary-rgb` | `139, 92, 246` | 用于 `rgba()` |
| `--accent` | `#f59e0b` | 金，境界/奖励/高阶元素 |

### 背景层次（四层）

| 变量 | 色值 | 用途 |
|------|------|------|
| `--bg-page` | `#0f0f1a` | 页面底 |
| `--bg-section` | `#151528` | 区域底（新增） |
| `--bg-card` | `#1a1a2e` | 卡片 |
| `--bg-input` | `#252540` | 卡片内输入区/二级区域 |

### 文字色

| 变量 | 色值 | 用途 |
|------|------|------|
| `--text` | `#e2e8f0` | 主文字 |
| `--text-dim` | `#94a3b8` | 次要文字 |
| `--text-bright` | `#f8fafc` | 高亮文字 |
| `--border` | `#334155` | 边框/分割线 |

### 功能色

| 变量 | 色值 | 用途 |
|------|------|------|
| `--success` | `#10b981` | 成功/良品（绿） |
| `--error` | `#ef4444` | 错误/衰退（红） |
| `--info` | `#3b82f6` | 信息/上品（蓝） |
| `--warning` | `#faad14` | 警告（黄） |

### 品质色映射

| 品质 | 色值 | 色名 |
|------|------|------|
| 凡品 | `#94a3b8` | 灰 |
| 良品 | `#10b981` | 绿 |

## 二、间距体系

两档间距，取代当前统一 12px。

| 场景 | 间距 |
|------|------|
| 同区域内元素间距 | `8px` |
| 跨区域/卡片间距 | `16px` |
| 页面左右 padding | `16px` |
| 卡片内 padding | `16px` |

---

## 三、圆角

```
--radius: 8px
```

12px 在深色主题下偏软，8px 更贴合修仙题材的硬朗感，与中国主流 app 的 8-12px 范围一致。

---

## 四、字号体系

| 用途 | 字号 | 字重 | 色值 |
|------|------|------|------|
| 页面大标题 | 20px | 700 | `--text-bright` |
| 卡片标题 | 15px | 600 | `--text-bright` |
| 主文字/按钮 | 15px（从 14px 提升） | 400-600 | `--text` |
| 副标题/说明 | 13px | 400 | `--text-dim` |
| 辅助信息/标签 | 12px | 400 | `--text-dim` |
| 底部导航 | 12px（从 11px 提升） | 400 | `--text-dim` / `--primary` |

规则：
- 行高 = 字号 × 1.5，向上取偶数
- 字重只用 400（regular）和 600（semibold）两档
- 700 仅用于页面大标题

---

## 五、触控热区

| 元素 | 规格 |
|------|------|
| 所有可交互元素最小触控区域 | 44×44px |
| 主按钮 `.btn` | min-height 44px（已满足） |
| 小按钮 `.btn-small` | 视觉高度 36px，padding 补到 44px 触控区 |
| 列表行高 | ≥ 48px |
| 底部导航单项 | 均分屏幕宽度（375px / 5 = 75px），高度 50px |

---

## 六、交互原则

- 核心交互只依赖 tap（点击）和 scroll（滚动）
- 下拉刷新作为辅助
- 不依赖左滑、长按、双击等进阶手势
- 所有可点击元素需要明确视觉提示（箭头、按钮样式、颜色区分）
- 按压反馈：active 态 `opacity: 0.8` 或背景色变化

---

## 七、组件规范

### 卡片 `.card`

```css
.card {
  background: var(--bg-card);       /* #1a1a2e */
  border-radius: var(--radius);     /* 8px */
  padding: 16px;
  border: 1px solid var(--border);  /* #334155 */
  /* 无阴影——深色主题下阴影无意义 */
}
```

### 底部导航

| 属性 | 值 |
|------|------|
| 高度 | `50px + env(safe-area-inset-bottom)` |
| 图标 | 20px |
| 文字 | 12px |
| 激活色 | `var(--primary)` |
| 非激活色 | `var(--text-dim)` |

### 成员网格

| 属性 | 值 |
|------|------|
| 布局 | 2 列网格 |
| 默认展示 | 6 人（3 行 × 2 列） |
| 头像 | 40px 圆形 |
| 昵称 | 12px 单行截断 |
| 境界 | 小标签 |
| 环境状态 | 头像右下角圆点（绿=居家，红=生病，蓝=出差） |
| 超出 | "+N 查看全部" |

### 动态 Feed 表情按钮

| 属性 | 值 |
|------|------|
| 排列 | 横向紧凑，只保留 emoji + 计数 |
| 按钮高度 | 36px（触控区 44px 通过 padding 补足） |
| 默认条数 | 10 条 + "加载更多" |

### 首页 Hero 区域

合并用户名 + 境界 + 修炼状态为一个区域，取代原来的两个独立 card：

- 第一行：用户名（左）+ 环境状态 badge（右）
- 第二行：境界 badge + 修炼状态标签 + 加成/缓冲标签
- 第三行（可选）：nextLevelHint 小字提示

---

## 八、设计原则

1. **信息密度适度放松**：相比中国主流 app 的紧凑布局，为中老年用户适度放松间距和字号，但不过度稀疏
2. **用背景层次分隔模块**：深色主题下用四层背景色建立视觉层次，而非依赖留白
3. **修仙氛围通过色彩而非装饰**：紫金配色 + 品质色阶传递修仙感，不堆砌纹理和装饰元素
4. **触控友好**：严格 44px 最小热区，照顾中老年用户
5. **点击为王**：不依赖进阶手势，所有操作通过 tap 完成
