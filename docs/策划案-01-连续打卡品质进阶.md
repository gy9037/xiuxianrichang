# 策划案-01：签到系统与灵石货币

> **状态**：已评审
> **评审日期**：2026-04-20
> **评审结论**：原方案不采纳（streak 影响行为掉落品质），重新设计为签到+货币系统
> **优先级**：P0（核心循环扩展）
> **关联系统**：行为系统、道具系统、货币系统（新增）
> **影响文件**：见第八节改动清单

---

## 一、背景与设计目标

### 评审结论

原方案设计为"连续打卡天数（streak）影响行为掉落道具品质"，评审后不采纳。原因：streak 直接绑定品质会让中断惩罚过重，用户一旦断签就失去高品质道具来源，容易产生挫败感。

评审决策：行为打卡的道具掉落维持现状不改动，新增独立的签到奖励系统和灵石货币系统，作为长期坚持的正反馈通道。

### 现状保留

当前 `determineQuality()` 的品质判定逻辑维持不变：

- 凡品/良品由强度+精进状态决定
- 身体健康类按运动强度确定基础良品率（热身10%、低强度20%、高强度40%、拉伸15%）
- 其他所有类固定 20% 良品率
- 精进状态叠加良品率 +10%
- 上品/极品不在行为掉落中产出（后续可通过商店、活动等渠道获取）

### 设计目标

1. **新增灵石货币**：为后续多货币体系打基础，灵石是第一种货币
2. **新增签到奖励**：让每日坚持有额外回报，连续签到奖励递增
3. **首页信息栏改造**：顶部去掉重复导航标题，改为 HUD 信息栏
4. **浮动功能入口**：右侧浮动按钮体系，当前只做签到，后续可扩展活动、商城等

## 二、灵石货币系统

### 2.1 设计原则

- 灵石（spirit_stones）是独立货币系统，不绑定签到逻辑，为后续多货币扩展留空间
- 灵石是第一种货币，后续可能有其他货币（如筹码）
- 货币表设计为通用结构，支持多币种

### 2.2 用途规划

| 阶段 | 用途 | 说明 |
|------|------|------|
| 当前版本 | 签到奖励发放 | 连续签到获得灵石 |
| 近期 | 补签卡购买 | 消耗灵石购买补签卡，恢复中断的签到 streak |
| 中期 | 商店 | 灵石兑换道具、外观等 |
| 远期 | 活动、付费悬赏 | 灵石参与活动奖励、悬赏任务等 |

### 2.3 灵石流水规则

每笔灵石变动记录流水，包含来源类型、数量、时间戳，便于审计和数据分析。

## 三、签到奖励系统

### 3.1 签到判定条件

当天（自然日，UTC+8）有任何行为记录即算签到成功。不需要用户手动签到，系统自动判定。

### 3.2 触发方式

自动触发：当天首次行为提交成功时，系统自动完成签到并发放灵石奖励。需要有明显特效让用户感知签到成功（弹窗 + 动效）。

### 3.3 连续签到灵石奖励

| 连续签到天数 | 灵石奖励 |
|-------------|---------|
| 1-5 天 | 1 灵石/天 |
| 6-10 天 | 2 灵石/天 |
| 11-20 天 | 3 灵石/天 |
| 21 天+ | 5 灵石/天 |

### 3.4 中断规则

- 中断后 streak 归零，从 1 天重新开始
- 暂无容错机制
- 未来扩展：补签卡（消耗灵石购买），补签后恢复 streak 连续性

### 3.5 签到数据示例

```
用户 A 连续签到情况：
第 1-5 天：每天 1 灵石，累计 5 灵石
第 6-10 天：每天 2 灵石，累计 5+10=15 灵石
第 11-20 天：每天 3 灵石，累计 15+30=45 灵石
第 21-30 天：每天 5 灵石，累计 45+50=95 灵石

连续签到 30 天总计可获得 95 灵石。
```

### 4.2 浮动按钮设计

## 七、数据库设计

### 7.1 新增表：`currencies`（货币余额表）

```sql
CREATE TABLE currencies (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  currency_type TEXT NOT NULL DEFAULT 'spirit_stones',  -- 货币类型，预留多币种
  balance INTEGER NOT NULL DEFAULT 0,                    -- 当前余额
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(user_id, currency_type)
);
```

### 7.2 新增表：`currency_transactions`（货币流水表）

```sql
CREATE TABLE currency_transactions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  currency_type TEXT NOT NULL DEFAULT 'spirit_stones',
  amount INTEGER NOT NULL,              -- 正数=收入，负数=支出
  source_type TEXT NOT NULL,            -- 'daily_checkin' | 'purchase' | 'activity' | 'admin'
  source_id TEXT,                       -- 关联来源 ID（如签到记录 ID）
  balance_after INTEGER NOT NULL,       -- 变动后余额（便于核对）
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_currency_tx_user ON currency_transactions(user_id, currency_type, created_at);
```

### 7.3 新增表：`checkin_records`（签到记录表）

```sql
CREATE TABLE checkin_records (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  checkin_date TEXT NOT NULL,            -- '2026-04-20'，自然日（UTC+8）
  streak INTEGER NOT NULL DEFAULT 1,    -- 当次签到时的连续天数
  reward_amount INTEGER NOT NULL,       -- 本次获得的灵石数
  trigger_behavior_id INTEGER,          -- 触发签到的行为记录 ID
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(user_id, checkin_date)
);
```

## 十、签到服务核心逻辑

```javascript
// checkinService.processCheckin(userId, behaviorId)
// 在行为提交成功后调用

async function processCheckin(userId, behaviorId) {
  const today = getTodayDateString(); // UTC+8 自然日

  // 1. 幂等检查：今日是否已签到
  const existing = await db.get(
    'SELECT * FROM checkin_records WHERE user_id = ? AND checkin_date = ?',
    [userId, today]
  );
  if (existing) {
    return { is_first_today: false, streak: existing.streak, reward: 0 };
  }

  // 2. 计算 streak：查询昨日是否有签到记录
  const yesterday = getYesterdayDateString();
  const yesterdayRecord = await db.get(
    'SELECT streak FROM checkin_records WHERE user_id = ? AND checkin_date = ?',
    [userId, yesterday]
  );
  const streak = yesterdayRecord ? yesterdayRecord.streak + 1 : 1;

  // 3. 根据 streak 计算奖励
  const reward = calculateReward(streak);

  // 4. 写入签到记录
  await db.run(
    'INSERT INTO checkin_records (user_id, checkin_date, streak, reward_amount, trigger_behavior_id) VALUES (?, ?, ?, ?, ?)',
    [userId, today, streak, reward, behaviorId]
  );

  // 5. 发放灵石
  const balance = await currencyService.addCurrency(userId, 'spirit_stones', reward, 'daily_checkin');

  return { is_first_today: true, streak, reward, spirit_stones_balance: balance };
}

function calculateReward(streak) {
  if (streak >= 21) return 5;
  if (streak >= 11) return 3;
  if (streak >= 6) return 2;
  return 1;
}
```
