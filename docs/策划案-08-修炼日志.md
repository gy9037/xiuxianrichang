# 策划案-08：修炼日志

> **状态**：暂缓
> **评审日期**：2026-04-20
> **评审结论**：暂缓实施。模板拼装生成的日志长期重复度高，代入感可能较弱。保留设计方案作为参考，后续视需要决定是否实施。
> **优先级**：P2
> **关联系统**：行为系统、道具系统、节气系统、境界系统
> **新增文件**：`server/services/diaryGen.js`
> **修改文件**：`server/routes/behavior.js`、`server/routes/character.js`

## 一、背景与设计目标

### 为什么做修炼日志

当前 App 的行为记录是**数据化**的：提交一次行为，返回一个道具名和属性值。用户看到的是 `+1.5 心性临时属性` 而不是一段故事。

这导致两个问题：

1. **每次打卡感受千篇一律**：不管今天做了什么，反馈都是 "获得 XX 丹（良品）"。坚持 30 天的用户和第 1 天的用户，主观感受几乎没有区别。
2. **时间感知缺失**：用户说不清"上周我修炼得怎么样"——不是因为没数据，而是没有自然语言叙述帮助记忆和回顾。

修炼日志的目标：
1. **把数据翻译成故事**：每天自动生成一段 2-3 句话的"修炼叙述"，让用户对当天的活动有真实感知
2. **制造阶段性成就感**：周结语、月结语提炼这段时间的成长，强化长期坚持的正反馈
3. **与修仙题材深度融合**：日志用第三人称叙事（"某某道友今日……"），沉浸感强，契合修仙主题

### 设计原则

- **完全自动生成**：用户零操作，行为数据自动变成日志文字
- **叙述感重于数据感**：日志不是行为列表的文字化，而是提炼后的叙事
- **可选阅读**：日志不强推，用户主动进入"修炼日志"页才看到，不占用主流程界面

## 二、日志层级设计

修炼日志分三个层级，自动触发，分别在不同时机生成：

| 层级 | 触发时机 | 内容 | 存储 |
|------|---------|------|------|
| **每日日志** | 当天有任意行为记录时，次日首次打开 App 可见 | 昨日修炼的叙事性总结 | `diary_entries` 表，type=daily |
| **周结语** | 每周一（UTC+8）自动生成上周总结 | 上周数据的主题提炼 + 成长评语 | `diary_entries` 表，type=weekly |
| **月结语** | 每月 1 日自动生成上月总结 | 上月最突出成就 + 属性变化 + 赛季感总结 | `diary_entries` 表，type=monthly |

> 特殊：冬至当日自动附加特殊冬至日志条目（节气系统联动）。

## 三、每日日志生成规则

### 3.1 触发条件

- 用户当天（自然日，UTC+8）有 ≥ 1 条行为记录
- 日志在次日首次访问 `/character` 或 `/diary` 接口时生成（懒生成，不依赖定时任务）
- 同一自然日最多生成一条每日日志（幂等）

### 3.2 日志结构

每日日志由 **开场句 + 行为描述句 + 收尾句** 三部分组成，共 2-4 句。

#### 开场句模板池（按当天总行为数选择）

| 当天行为数 | 模板示例 |
|-----------|---------|
| 1 条 | "今日修炼，{角色名}略得小成，踏出一步。" |
| 2-3 条 | "今日{角色名}潜心修炼，勤耕不辍。" |
| 4-5 条 | "今日{角色名}大修，灵气充盈，诸事皆进。" |
| ≥ 6 条 | "今日{角色名}修炼繁忙，道心坚定，颇为难得。" |

#### 行为描述句（每个涉及的属性系生成一句）

| 属性 | 模板示例 |
|------|---------|
| 体魄 | "身体力行，{行为描述}，筋骨得以淬炼。" |
| 悟性 | "研习{行为描述}，思路渐明，悟性有所增益。" |
| 心性 | "{行为描述}，心绪沉稳，道心愈固。" |
| 灵巧 | "动手{行为描述}，手法精进，灵巧小有所成。" |
| 神识 | "{行为描述}，感知细腻，神识得以锻炼。" |

> `{行为描述}` 取当天该属性类别得分最高的行为的 `sub_type`，如"早起"、"读书"、"跑步"。

#### 收尾句（按道具品质最高者选择）

| 最高品质 | 模板示例 |
|---------|---------|
| 凡品 | "今日所获寻常，却是修炼的基石。" |
| 良品 | "今日修炼有成，获一良品道具，可以一用。" |
| 上品 | "今日坚持有功，所获上品道具，灵气充沛。" |
| 极品 | "今日修炼至深处，竟感应天地，获极品道具，实属难得！" |

#### 特殊触发句（满足条件时插入，最多插入一条）

| 触发条件 | 插入句示例 |
|---------|-----------|
| 今日有节气加成 | "此时正值{节气名}，天地灵气特别浓郁。" |
| 今日 streak 整数周（7、14、21...） | "连续修炼{N}日，道心颇为稳固。" |
| 今日突破境界 | "今日感悟深透，境界有所突破，踏入{新境界}。" |
| 今日家庭多人共同有记录（≥3人） | "今日全家同修，道气相合，颇为和谐。" |
| 今日获得节气限定道具 | "今日意外感应，获得节气限定道具「{道具名}」。" |

### 3.3 日志示例

**场景**：用户"李鸿远"（道号：云鹤居士），当天记录了早起、读书、跑步，全部都是良品，当前节气为立秋。

```
今日云鹤居士潜心修炼，勤耕不辍。

身体力行，晨跑锻炼，筋骨得以淬炼。研习读书，思路渐明，悟性有所增益。心绪沉稳于早起之间，道心愈固。

此时正值立秋，天地灵气特别浓郁。今日修炼有成，获一良品道具，可以一用。
```

## 四、周结语生成规则

### 4.1 触发条件

- 每周一（UTC+8 00:00-01:00）之间，由定时任务扫描所有上周有记录的用户
- 或：用户当周首次打开 App 时懒生成（推荐，省资源）

### 4.2 内容结构

周结语由 **本周评语 + 亮点项 + 成长句 + 建议方向** 四段组成。

**本周评语**（按本周总行为天数）：

| 活跃天数 | 评语方向 |
|---------|---------|
| 0 天 | 不生成（该用户本周无记录，跳过） |
| 1-2 天 | "此番修炼略显散漫，但踏出第一步总是好的" |
| 3-4 天 | "本周修炼尚算勤勉，已见道心萌动" |
| 5-6 天 | "本周修炼颇为勤奋，灵台清明，进境稳健" |
| 7 天 | "本周七日无间断，修炼意志堪称道侣之中翘楚！" |

**亮点项**（自动选取本周表现最突出的一项）：

规则：选取本周 streak 最长的 sub_type，或道具品质最高的行为。格式示例：
> 「早起」连续坚持 12 天，品质稳定在上品，灵气充盈。

**成长句**（基于本周属性净增量）：

> 本周{属性增量最大的属性名}增长最为显著，临时属性积累 +{X}，炼化后可增益明显。

**建议方向**（基于本周最少涉及的属性）：

> {最少涉及属性}方面本周稍有欠缺，下周可适当兼顾，以期全面精进。

### 4.3 周结语示例

```
本周修炼颇为勤奋，灵台清明，进境稳健。

【本周亮点】
「读书」连续坚持 9 天，品质稳定在上品，悟性灵气充盈。

【本周成长】
本周悟性增长最为显著，临时属性积累 +8.5，可炼化为 +1 永久属性。

【下周方向】
灵巧方面本周稍有欠缺，下周可适当兼顾，以期五行均衡精进。
```

## 五、月结语生成规则

### 5.1 内容结构

月结语是最重要的阶段性总结，内容最丰富：

1. **月度评语**：本月总体修炼状态（勤奋度 + 稳定性综合评价）
2. **最长 Streak 记录**：本月坚持最久的行为 + 天数
3. **属性成长表**（简表，文字版）：5 个属性各增长了多少
4. **境界变化**：本月是否突破境界
5. **限定道具获得情况**：本月获得了几种节气限定道具（如果有节气在本月内）
6. **一句话赛季评语**：有个性的总结收尾

### 5.2 月结语示例

```
三月修炼总录

本月云鹤居士勤耕不辍，修炼二十一日，五行均有涉猎，道心稳固可嘉。

【最长坚持】
「早起」本月坚持 18 天连续不断，堪称本月修炼之基石。

【属性成长】
体魄 +2 ｜ 悟性 +3 ｜ 心性 +1 ｜ 灵巧 +2 ｜ 神识 +1

【境界变化】
本月于惊蛰之时突破至练气五阶，道法精进，可喜可贺。

【限定道具】
本月历经惊蛰、春分两个节气，共获得限定道具 2 种：
春雷惊蛰符 · 阴阳平衡诀

一句话总结：春风化雨，正是苦修好时节。
```

## 六、展示方案

### 6.1 入口位置

在**角色页**底部新增「修炼日志」入口按钮，点击进入日志列表页（不占 Tab 位）。

```
┌────────────────────────────────────┐
│  [境界信息] [属性栏]               │
│  ─────────────────────────────── │
│  📖 修炼日志  →  查看修炼历程      │
└────────────────────────────────────┘
```

### 6.2 日志列表页

日志按时间倒序排列，分组展示：

```
── 本月 ──────────────────────────────
  [月结语卡片]  4月月度总结  ▶
  [日记卡片]   4月17日修炼记录  ▶
  [日记卡片]   4月16日修炼记录  ▶
  [日记卡片]   4月15日修炼记录  ▶
  ...

── 上周 ─────────────────────────────
  [周结语卡片]  第15周修炼总结  ▶
  ...
```

- 月结语和周结语用不同颜色卡片区分（月=金色，周=蓝色，日=默认）
- 每条日志卡片显示：日期 + 首句预览（30字以内）
- 点击进入完整日志详情页

### 6.3 新日志提示

当存在用户未读的日志时（尤其是周结语/月结语），在**角色页**的「修炼日志」入口旁显示红点提示。

## 七、数据库设计

### 新增表：`diary_entries`

```sql
CREATE TABLE diary_entries (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  type TEXT NOT NULL,          -- 'daily' | 'weekly' | 'monthly'
  period_key TEXT NOT NULL,    -- '2026-04-17' | '2026-W15' | '2026-04'
  content TEXT NOT NULL,       -- 生成的日志文字（JSON 结构存储）
  is_read INTEGER DEFAULT 0,   -- 0=未读, 1=已读
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(user_id, type, period_key)
);
```

`content` 字段存储 JSON，结构示例：
```json
{
  "title": "四月十七日修炼记录",
  "paragraphs": [
    "今日云鹤居士潜心修炼，勤耕不辍。",
    "身体力行，晨跑锻炼，筋骨得以淬炼。",
    "此时正值立秋，天地灵气特别浓郁。",
    "今日修炼有成，获一良品道具，可以一用。"
  ],
  "highlights": {
    "max_quality": "良品",
    "behaviors_count": 3,
    "solar_term": "立秋"
  }
}
```

## 九、数值与生成策略说明

- 日志文字完全由模板 + 用户数据拼装，**不调用 LLM API**，零成本，毫秒级生成
- 模板库初始版本约 50-80 条，后续可扩充
- 生成结果存储于数据库，不重复生成（幂等保证）
- 长期规划（V2.0+）：可选接入 AI 文案增强，将模板句升级为更自然的叙事，但核心功能不依赖 AI
