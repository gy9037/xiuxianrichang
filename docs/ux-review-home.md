# UX Review — 首页 (home.js)

> 审查时间：2026-04-09
> 用户群：家庭（含老人小孩），移动端为主
> 审查维度：操作步骤、即时反馈/loading、视觉引导、移动端适配、状态一致性

---

## 问题清单

| 问题描述 | 严重度 | 建议修复方向 |
|---|---|---|
| `load()` 期间无任何 loading 状态，数据未到前页面空白，老人/小孩容易误以为卡死 | 🔴 | 进入首页时立即渲染骨架屏或 spinner，数据返回后替换 |
| `promote()` 点击后无 loading 反馈，按钮可被重复点击，可能触发多次晋级请求 | 🔴 | 点击后立即禁用按钮并显示"晋级中…"，请求完成后恢复 |
| `setStatus()` 提交期间弹窗无 loading，用户可连续点击不同状态选项，导致竞态 | 🔴 | 提交时禁用所有选项并显示加载指示，完成后关闭弹窗 |
| 状态 badge 点击区域仅靠文字宽度，移动端触控目标过小（< 44px），老人难以精准点击 | 🔴 | 增大 padding 至至少 44×44px 触控区，或用独立图标按钮 |
| 推荐卡片整体可点击（`onclick` 在外层 div），但无任何视觉按压/hover 反馈，用户不知道它可交互 | 🟡 | 添加 `active:opacity-80` 或 CSS `:active` 压暗效果，加箭头颜色变化 |
| 衰退预警仅用文字堆叠，无颜色区分严重程度，老人/小孩难以快速识别紧急程度 | 🟡 | 按衰退状态（警告/严重）加色块或左侧色条区分，配合图标 |
| 雷达图 SVG 固定 200×200px，在小屏（< 360px）或大屏平板上不自适应，比例失调 | 🟡 | 改用 `viewBox` + `width:100%` 响应式布局，移除固定 width/height 属性 |
| 成就 toast 在页面加载时批量弹出（`forEach` 逐条），多条成就同时触发会堆叠遮挡内容 | 🟡 | 改为队列式逐条延迟弹出（每条间隔 1.5s），或合并为"解锁了 N 个成就" |
| 新用户引导推荐卡（"先去上报一次行为"）整个卡片绑定 `onclick`，但视觉上看不出可点击，缺乏 CTA 按钮 | 🟡 | 改为明确的按钮样式（如"去上报 →"），而非隐式整卡点击 |
| `goToBehavior` 用 `setTimeout(50ms)` 等待 BehaviorPage 渲染后预选类别，时序脆弱，低端机可能失效 | 🟡 | 改为事件/回调机制：BehaviorPage 渲染完成后主动消费待选类别参数 |
| 退出登录仅为一行小字，无确认步骤，老人误触后直接登出，体验差 | 🟡 | 点击后弹出确认对话框，或改为设置页入口，降低误触风险 |
| 属性进度条（`attr-bar-fill`）宽度通过内联 style 注入，无过渡动画，数值变化时无视觉反馈 | 🟢 | 添加 `transition: width 0.4s ease`，让进度条变化有动画 |
| 境界 badge 可晋级时样式为 `promotable` 按钮，不可晋级时为普通 `span`，两者视觉差异大但语义未区分，屏幕阅读器无法识别 | 🟢 | 不可晋级的 span 加 `role="status"` 或 `aria-label`，可晋级按钮加 `aria-label="点击晋级"` |
| 成就列表无空状态提示（`achievements.length === 0` 时直接返回空字符串），用户不知道成就系统存在 | 🟢 | 空状态时渲染"暂无成就，继续修炼解锁"占位提示 |
| 状态弹窗通过 `document.createElement` 动态插入，关闭仅靠 `remove()`，点击遮罩层无法关闭，移动端体验不友好 | 🟢 | 给遮罩层本身绑定 `onclick` 关闭，或统一使用 `<dialog>` 元素 |
