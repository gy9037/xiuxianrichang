## 一、背景与设计目标

### 核心定位

擂台系统是即时竞技系统，与任务系统互补：

| 系统 | 时间维度 | 参与方式 | 示例 |
|------|---------|---------|------|
| 任务系统 | 长期目标（天/周/月） | 异步完成 | 本周跑步5次、读完一本书 |
| 擂台系统 | 即时竞技（当场发生） | 同步参与 | 投篮比赛、出题挑战、打麻将 |

### 设计原则

- 即时性：擂台是当场发生的活动，不是长期任务
- 轻量化：发起和参与流程尽量简短
- 多场景：覆盖智力、体能、娱乐三种家庭互动场景

---

## 二、三种擂台类型

### 2.1 出题挑战

A 解开一道题或完成一件事，发到擂台让大家做。出题者判定对错。

- 发起者：出题并设定答案/判定标准
- 参与者：提交答案或完成证据
- 判定：出题者判定对错
- 奖励：系统奖池发放灵石

### 2.2 对局记录

麻将、桌游等线下娱乐活动的记录工具。

- 记录内容：起始时间、结束时间、参与者、结果登记
- 不做记分板（线下有实时记录）
- 货币结算：支持货币选择
  - 灵石：通用货币
  - 筹码：麻将专用独立货币，参与者之间流转（赢家从输家处获得）

### 2.3 体能比拼

投篮、俯卧撑、平板撑等体能项目。

- 各自提交成绩（数字+可选照片/视频证据）
- 系统自动排名
- 奖励：系统奖池发放灵石

---

## 三、独立货币：筹码

| 属性 | 说明 |
|------|------|
| 名称 | 筹码 |
| 用途 | 麻将等对局场景专用 |
| 与灵石关系 | 完全独立，不可兑换 |
| 流转方式 | 参与者之间流转（零和） |
| 初始分配 | 每人初始筹码数待定 |

---

## 四、证据提交

- 支持提交图片作为证据，但不强制
- 复用任务系统的图片上传能力（R2 存储）
- 体能比拼场景下，照片/视频可增加可信度

## 五、数据库设计

### 新增表：`arenas`

```sql
CREATE TABLE arenas (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  family_id INTEGER NOT NULL,
  creator_id INTEGER NOT NULL,
  type TEXT NOT NULL,              -- 'quiz'|'match'|'fitness'
  title TEXT NOT NULL,
  description TEXT,
  config TEXT,                     -- JSON: 类型特定配置
  currency TEXT DEFAULT 'stones',  -- 'stones'|'chips'
  reward_pool INTEGER DEFAULT 0,  -- 系统奖池灵石数
  status TEXT DEFAULT 'active',   -- 'active'|'completed'|'cancelled'
  started_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  completed_at DATETIME
);
```

### 新增表：`arena_participants`

```sql
CREATE TABLE arena_participants (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  arena_id INTEGER NOT NULL,
  user_id INTEGER NOT NULL,
  submission TEXT,                 -- JSON: {score, text, photo_urls}
  result TEXT,                     -- 'win'|'lose'|'draw'|null
  currency_change INTEGER DEFAULT 0, -- 货币变动（正为获得，负为失去）
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(arena_id, user_id),
  FOREIGN KEY (arena_id) REFERENCES arenas(id)
);
```

## 六、API 设计

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/arenas | 擂台列表 |
| POST | /api/arenas | 创建擂台 |
| POST | /api/arenas/:id/join | 加入擂台 |
| POST | /api/arenas/:id/submit | 提交成绩/答案 |
| POST | /api/arenas/:id/judge | 出题者判定（仅 quiz 类型） |
| POST | /api/arenas/:id/settle | 结算（对局记录登记结果） |
| GET | /api/arenas/:id | 擂台详情 |
