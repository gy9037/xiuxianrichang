# UX Review — 背包页（inventory.js）

> 审查日期：2026-04-09
> 用户群：家庭（含老人小孩），移动端为主
> 审查维度：操作步骤、即时反馈/loading、视觉引导、容错性、移动端适配、状态一致性

---

## 问题清单

| 问题描述 | 严重度 | 建议修复方向 |
|---|---|---|
| `synthesize()` 和 `redeem()` 无 loading 状态，按钮点击后无任何视觉反馈，网络慢时用户会重复点击 | 🔴 | 点击后立即禁用按钮并显示"合成中…"/"兑现中…"文字，请求完成后恢复 |
| `redeem()` 无二次确认，老人/小孩误触即触发不可逆操作 | 🔴 | 弹出确认对话框："确认将此奖励标记为已兑现？"，需主动确认才执行 |
| `synthesize()` 无二次确认，道具消耗不可逆，且余数会被浪费，用户可能在不理解规则的情况下误操作 | 🔴 | 合成前弹窗展示"将消耗 X 件道具，获得永久属性 +N，浪费 Y 点，确认？" |
| `load()` 失败时只有 toast 提示，页面内容为空白，用户不知道是网络问题还是真的没有数据 | 🔴 | catch 块中渲染错误占位页（含重试按钮），区分"加载失败"与"背包为空" |
| `toggleItem()` / `selectAll()` / `selectNone()` 每次都调用 `render()` 重绘整个页面，选中状态变化时页面闪烁，移动端体验差 | 🟡 | 仅更新 checkbox 状态和底部合成摘要栏，避免全量重绘 |
| 合成摘要栏（`synth-summary`）固定在页面流中，滚动后不可见，用户选好道具后找不到合成按钮 | 🟡 | 将摘要栏改为 `position:fixed` 吸底浮层，始终可见 |
| 分类 Tab 横向滚动区域（`overflow-x:auto`）无滚动指示，用户不知道右侧还有更多分类 | 🟡 | 右侧添加渐变遮罩或">"箭头提示，或在 Tab 下方加滚动条指示点 |
| 道具行点击区域仅限 checkbox（约 20×20px），老人/小孩难以精准点击 | 🟡 | 将整个 `item-row` 设为可点击区域（`onclick="InventoryPage.toggleItem(...)"`），提升触控目标尺寸至 44px 以上 |
| `showSynthesisRule()` 弹窗无法通过点击背景关闭，只能点"明白了"按钮，操作路径单一 | 🟡 | 给遮罩层添加 `onclick` 关闭事件，支持点击背景关闭 |
| 合成规则弹窗首次自动弹出（`setTimeout 300ms`），但 `load()` 是异步的，数据未加载完时弹窗可能覆盖空白页面，时序混乱 | 🟡 | 将首次弹窗移到 `render()` 完成后触发，或在数据渲染完毕的回调中执行 |
| "浪费 X 点"提示字号仅 11px，颜色为 `text-dim`，老人用户几乎不可见，但这是重要的损耗提示 | 🟡 | 字号提升至 13px，颜色改为警示色（如橙色），或在合成确认弹窗中突出显示 |
| 切换 Tab（`switchTab`）会清空 `selectedIds`，但切换 Section（`switchSection`）不清空，两者行为不一致 | 🟡 | 统一策略：切换 Section 时也清空选中状态，或两者都保留（并在 UI 上提示跨 Tab 选中不生效） |
| 奖励列表中"发起人"字段直接显示（`creator_name`），无角色说明，家庭中老人/小孩不理解"发起人"含义 | 🟢 | 改为"由 XX 发起"或添加 tooltip 说明角色关系 |
| 已兑现奖励列表无时间戳，用户无法回溯兑现历史 | 🟢 | 在已兑现记录中展示兑现时间（如"2026-04-08 兑现"） |
| 背包为空时引导文案"完成行为上报可获得道具"过于简短，新用户不知道如何操作 | 🟢 | 添加跳转链接或按钮，直接引导至行为上报页 |
| `quality-${q}` 样式中对非法品质值做了兜底（默认凡品），但 `item-meta` 仍会显示原始非法品质字符串，数据异常时用户看到奇怪文字 | 🟢 | 对品质字段做统一 normalize，或在显示前过滤非法值 |
