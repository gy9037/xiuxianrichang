# UX 审查报告 Part 2：背包页 / 愿望页 / 家庭页

> 审查时间：2026-04-10
> 用户群：家庭（含老人小孩），移动端为主
> 审查文件：inventory.js / wish.js / family.js

---

## 一、背包页（inventory.js）

### 1. 合成操作无二次确认，道具消耗不可逆
点击「合成」按钮直接调用 API，没有确认弹窗。道具合成后消失且余数浪费，老人小孩容易误触。
| 🔴严重 | 合成前弹出确认弹窗，明确展示"将消耗 X 件道具，浪费 Y 点，获得 +Z 永久属性"，需用户确认后执行。

### 2. 兑现奖励无二次确认
`redeem()` 直接 POST，没有确认步骤。奖励兑现是一次性操作，误触后无法撤回。
| 🔴严重 | 增加确认弹窗："确认兑现「XXX」？兑现后不可撤回"。

### 3. 合成/兑现操作无 loading 状态
`synthesize()` 和 `redeem()` 在 await 期间按钮仍可点击，可能导致重复提交。
| 🟡中等 | 请求发出后立即 disable 按钮并显示 loading 态（如"合成中..."），完成后恢复。

### 4. 每次勾选/取消道具都触发全量 render()
`toggleItem()` → `render()` 会重建整个 DOM，道具多时页面闪烁，checkbox 焦点丢失，滚动位置重置。
| 🟡中等 | 改为局部更新：只更新底部合成摘要栏和对应 checkbox 状态，不重建整个列表。

### 5. 道具 tab 横向滚动区无视觉提示
tab 容器设置了 `overflow-x:auto` 但没有滚动指示器，用户不知道右侧还有更多分类。
| 🟡中等 | 添加右侧渐变遮罩或滚动箭头提示，暗示可横向滑动。

### 6. checkbox 触摸目标过小
原生 `<input type="checkbox">` 在移动端点击区域约 16×16px，老人小孩难以精准点击。
| 🟡中等 | 将整个 `item-row` 设为可点击区域，或用自定义 checkbox 放大触摸热区至 44×44px（WCAG 最低要求）。

### 7. 合成摘要栏（synth-summary）可能被底部导航遮挡
选中道具后底部出现的浮动摘要栏没有设置 `bottom` 安全距离，在有底部导航栏的移动端可能被遮挡。
| 🟡中等 | 为 synth-summary 添加 `padding-bottom` 或 `margin-bottom`，预留底部导航高度。

### 8. 空背包状态缺少行动引导按钮
空状态只有文字"完成行为上报可获得道具"，没有跳转入口。
| 🟢轻微 | 增加"去上报"按钮，直接跳转到行为上报页面，减少用户寻路成本。

## 二、愿望页（wish.js）

### 9. 「开始挑战」按钮无二次确认，道具战后消耗不可逆
`executeBattle()` 直接发起战斗，装备的道具"无论输赢都会消耗"。用户可能没注意到已勾选的道具就点了挑战。
| 🔴严重 | 点击后弹出确认："即将消耗 X 件道具进入战斗，无论胜负道具均不返还，确认挑战？"

### 10. 愿望兑现无二次确认
`redeem()` 直接 POST，与背包页同样的问题。
| 🔴严重 | 同 #2，增加确认弹窗。

### 11. 战斗准备阶段无 loading 骨架屏
`startBattle()` 先渲染空页面再请求数据，Boss 区域只显示"正在推演Boss天机..."纯文字，用户不确定是否在加载。
| 🟡中等 | 用骨架屏或 spinner 动画替代纯文字，给出明确的加载进度感。

### 12. executeBattle 期间无 loading / 防重复点击
「开始挑战！」按钮在 await 期间仍可点击。
| 🟡中等 | 请求发出后 disable 按钮，显示"战斗中..."，防止重复提交。

### 13. 创建愿望表单无草稿保存
填写到一半误触返回（← 按钮），`closeCreate()` 直接清空所有输入，无任何提示。
| 🟡中等 | 返回前检测表单是否有内容，有则弹出"放弃编辑？"确认；或用 sessionStorage 暂存草稿。

### 14. 战斗回合动画无跳过机制
回合以 600ms 间隔逐条展示，5 回合需要 3 秒。重复挑战时用户可能不想每次都看动画。
| 🟡中等 | 添加"跳过动画"按钮，一次性展示所有回合。

### 15. 筛选器状态在页面切换后丢失
`typeFilter` 和 `statusFilter` 存在内存中，切换到其他页面再回来后 `load()` 不会重置，但如果页面对象被重建则丢失。行为不一致。
| 🟢轻微 | 明确策略：要么每次进入重置为"全部"，要么用 sessionStorage 持久化筛选状态。

### 16. 战斗页道具列表无分类/搜索
所有道具平铺展示，道具多时难以找到目标道具。
| 🟢轻微 | 按属性类型分组展示，或提供简单的属性筛选 tab。

### 17. 难度滑块缺少触觉反馈
range input 在移动端拖动时没有步进吸附感，1-10 的精度在小屏上难以精确选择。
| 🟢轻微 | 改用 10 个可点击的数字按钮或星级选择器，比滑块更适合移动端粗粒度选择。

## 三、家庭页（family.js）

### 18. 表情互动后全量重新加载页面
`react()` 调用 `this.load()` 重新请求 3 个 API 并全量 render，只为更新一个表情计数。页面闪烁，滚动位置丢失。
| 🔴严重 | 改为乐观更新：点击后立即更新本地计数和高亮状态，后台静默发请求。失败时回滚并 toast 提示。

### 19. 动态列表无分页/虚拟滚动
`feed` 一次性全量渲染，家庭活跃时动态可能很多，导致页面卡顿。
| 🟡中等 | 初始只加载最近 20 条，底部添加"加载更多"按钮或无限滚动。

### 20. 表情按钮无点击反馈
点击表情按钮后在 API 返回前没有任何视觉变化，用户不确定是否点击成功。
| 🟡中等 | 点击后立即切换高亮状态（乐观更新），配合轻微缩放动画。

### 21. 成员属性总和缺少上下文
只显示"属性总和 XX"，新用户不知道这个数字意味着什么，也无法查看各属性明细。
| 🟡中等 | 点击成员行展开属性雷达图或明细列表，让数字有意义。

### 22. 团队愿望进度缺少视觉进度条
团队愿望只用文字"XX:已通过 · YY:待挑战"展示进度，不够直观。
| 🟢轻微 | 添加简单的进度条或成员头像 + 勾选状态的可视化展示。

### 23. 家庭动态时间格式不友好
使用 `toLocaleString()` 输出完整日期时间，信息密度高但不易快速扫描。
| 🟢轻微 | 改为相对时间（"3分钟前""昨天"），hover/长按时显示完整时间。

### 24. 页面加载失败只有 toast，无重试入口
`load()` 的 catch 只弹 toast，页面停留在空白状态，用户不知道如何恢复。
| 🟡中等 | 加载失败时显示错误状态页，带"重新加载"按钮。此问题同样存在于背包页和愿望页。

## 优先修复建议

1. 第一优先级：所有不可逆操作增加二次确认（#1, #2, #9, #10）
2. 第二优先级：表情互动改为乐观更新（#18），所有异步操作增加 loading + 防重复提交（#3, #12）
3. 第三优先级：全量 render 改局部更新（#4），加载失败增加重试（#24）
