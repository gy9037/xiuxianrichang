## 一、背景与设计目标

### 为什么做节气系统

当前的 App 体验是**时间无感**的——每天打卡、偶尔炼化、偶尔打 Boss，内容完全均匀分布，没有任何周期性的节奏变化。用户感受不到"这段时间很特别，要好好修炼"。

节气系统的目标：
1. **创造时间节点感**：每 15 天一个轻微的内容变化，让用户有"原来已经到芒种了"的时间流动感
2. **引导行为多样化**：不同节气关联不同属性，自然引导用户在不同时期尝试不同类型的行为
3. **与修仙题材深度融合**：二十四节气本身就是中国传统文化中感知天道的方式，和修仙主题天然契合

### 设计原则

- **轻量**：节气事件不需要用户主动操作，自动生效，感知但不打扰
- **无惩罚**：节气只提供加成，不因未做某类行为而惩罚用户
- **真实日期**：完全对应真实二十四节气的日期，不虚构

### 2.2 每个节气的结构

```json
{
  "id": "liqiu",
  "name": "立秋",
  "date_mmdd": "0807",
  "date_end_mmdd": "0821",
  "attributes": ["comprehension"],
  "theme": "秋收启智",
  "flavor": "金风送爽，天地间灵气随之收敛，正是参悟道法的最佳时机。",
  "effect": {
    "type": "quality_boost",
    "target_categories": ["学习"],
    "boost": "streak_threshold_minus_1"
  },
  "limited_item": {
    "name": "金秋悟道露",
    "quality": "良品",
    "attribute": "comprehension"
  },
  "special": false
}
```

### 2.3 节气效果类型

| 效果类型 | 说明 | 适用节气示例 |
|---------|------|------------|
| `quality_boost` | 关联行为类别品质判定提升（streak 阈值 -1 天） | 大多数节气 |
| `double_item` | 关联类别行为额外产出 1 件凡品道具 | 芒种、小满 |
| `guaranteed_liang` | 关联类别首次行为必出良品 | 大暑（体魄强化） |
| `all_boost` | 所有属性品质判定提升 | 冬至（特殊） |
| `exploration_bonus` | （若秘境系统上线）秘境事件成功率提升 | 春分 |

## 三、冬至特殊设计

冬至作为**年度最强节气**，单独设计：

| 维度 | 设计 |
|------|------|
| **持续时间** | 当日 +3 天（共 4 天，比普通节气长） |
| **效果** | 全部 5 个属性类别的行为，品质判定 streak 阈值均降低 2 天 |
| **限定道具** | 「天地归元丹」— 极品，唯一可能出现极品的**非 streak** 途径（冬至期间所有行为有 3% 概率出极品） |
| **家庭联动** | 若全家所有成员在冬至当天都有行为记录，触发「全家同修冬至」事件，家庭动态发出特殊公告 |
| **修炼日志** | 自动生成特殊冬至日志条目："冬至一阳生，修炼之路迎来新的轮回。" |

## 六、节气日期算法

节气日期每年浮动 1-2 天，不能硬编码。后端使用算法计算：

### 方案一：预计算表（推荐，简单可靠）

在 `solar-terms.json` 中预存 2025-2030 年每个节气的精确日期，后端读取即可。约 130 行 JSON，维护成本极低。

```json
{
  "2026": {
    "liqiu": "2026-08-07",
    "chushu": "2026-08-23",
    ...
  }
}
```

### 方案二：实时计算

使用寿星万年历算法动态计算，精确但实现复杂。对于本项目不必要。

## 八、数值平衡说明

- 节气加成仅持续 15 天，不会长期影响属性增长速度
- 即使拿到更多良品，炼化效率提升有限（良品 1.5 vs 凡品 1.0，提升 50%）
- 限定道具和普通道具品质相同（仅名称不同），不影响数值
- 冬至特殊的极品出现概率（3%）极低，不会造成数值崩溃
