## 一、需求来源索引

本版本需求来源于 2026-04-07/08 与用户的讨论，原始反馈如下：

| 编号 | 原始反馈摘要 |
|------|------------|
| FB-01 | 新手引导几乎为零，登录后不知道该做什么 |
| FB-02 | 道具合成门槛不透明，不知道合成代价和浪费原因 |
| FB-03 | 属性衰退机制太隐性，出差/生病后回来发现属性掉了体验差 |
| FB-04 | Boss战缺乏预期，挑战前不知道胜率，失败后没有方向建议 |
| FB-05 | 行为上报流程步骤偏多，重复行为每次都要点4-5步 |
| FB-06 | 家庭页面缺乏互动感，没有点赞、评论、鼓励等手段 |
| FB-07 | 境界名称对老人不友好；愿望难度滑块缺乏参考标准 |
| FB-08 | 用户主动提出：用雷达图展示5个属性数值 |
| FB-09 | 用户主动提出：BUFF状态系统（生病/出差等状态切换行为规则） |

## 二、V2 大版本方向

**让游戏真正好玩、好用。**

核心闭环已有，这个阶段把每个环节的体验做扎实。不追求功能数量，追求每个功能用起来舒服。

## 三、功能清单

### P0 — 体验地基

#### V2-F01：行为上报简化
- **来源**：[FB-05]
- **解决的问题**：高频操作摩擦太大，重复行为每次都要点4-5步，影响日活
- **设计方向**：
  - 增加"常用行为"快捷入口，展示个人 Top5 高频行为
  - 点击常用行为直接跳到确认步骤（跳过类别选择）
  - 支持一键重复上次同类行为
  - 目标：重复行为压缩到2步以内

#### V2-F02：道具合成透明化
- **来源**：[FB-02]，关联 [REQ-G3]
- **解决的问题**：核心规则不可见，玩家不知道合成代价，不知道为什么有浪费
- **设计方向**：
  - 合成界面增加规则说明卡片（首次进入弹出，之后可随时查看）
  - 道具详情页标注"当前可合成X点永久属性"
  - 已消耗道具给出明确的"已转化"状态，而不是直接消失
  - 合成预览：选中道具后实时显示"将获得X点永久属性，浪费Y点"

#### V2-F03：情境化引导
- **来源**：[FB-01]，用户设计思路见 [DIS 2026-04-07 讨论]
- **解决的问题**：没有引导等于没有入口，新用户无法自主跑通闭环
- **设计方向**：
  - 不做一次性教程，做持续的"当前最优行为推荐"
  - 根据属性短板、最近7天行为记录、当前时间段，在首页展示1-2条"今日推荐行为"
  - 推荐逻辑初期用简单规则驱动（某属性低于均值30%则推荐对应行为类别）
  - 推荐卡片可直接跳转到对应上报流程

### P1 — 体验质量与家庭粘性

#### V2-F04：属性衰退/BUFF状态系统（最小版本）
- **来源**：[FB-03]，用户设计思路见 [DIS 2026-04-07 讨论]，关联 [REQ-C1]
- **解决的问题**：衰退机制是隐性惩罚，出差/生病后回来发现属性掉了体验差
- **设计方向（最小版本）**：
  - 引入角色状态标签：正常 / 生病 / 出差 / 休假
  - 由家庭管理员手动切换成员状态
  - 不同状态下首页展示对应推荐行为集合（生病→吃药忌口，出差→打卡分享）
  - 状态期间衰退规则宽松化（暂停或减缓衰退）
  - 行为规则联动（状态专属行为）留待后续版本完善

#### V2-F05：Boss战体验优化
- **来源**：[FB-04]，关联 [REQ-D3]、[REQ-G1]
- **解决的问题**：闭环终点没有重量感，挑战前无预期，失败后无方向
- **设计方向**：
  - 挑战前展示"胜算几成"（基于当前属性vs Boss需求的对比，修仙风格文案包装）
  - 失败后给出差距分析和1-2条具体提升建议（如"体魄差距最大，建议本周多做运动类行为"）
  - 胜利时增加仪式感演出（动画/音效）并推送家庭通知

#### V2-F06：家庭页互动
- **来源**：[FB-06]，关联 [REQ-H1]
- **解决的问题**：家庭向游戏的核心粘性来源，纯展示等于放弃了社交价值
- **设计方向**：
  - 为每条行为上报记录增加点赞和修仙风格表情回应（限定几个，降低门槛）
  - 支持简短评论（可选，不强制）
  - 行庭成员行为动态按时间流展示，形成"家庭日志"感

#### V2-F07：行为历史记录
- **来源**：策划补充，关联 [REQ-F1]
- **解决的问题**：玩家看不到自己的成长轨迹，削弱养成感
- **设计方向**：
  - 增加"行为日历"视图，按日期展示历史上报记录
  - 增加"本周总结"：本周完成行为数、获得道具数、属性变化
  - 作为家庭互动的内容基础

### P2 — 体验细节打磨

#### V2-F08：属性雷达图
- **来源**：[FB-08]
- **解决的问题**：5个属性用数字展示缺乏直觉感，养成感不强
- **设计方向**：
  - 角色主页增加五维雷达图，实时反映当前属性分布
  - 图形变化本身就是成长反馈
  - 雷达图形状可作为角色"个性"的视觉标识

#### V2-F09：境界与难度文案优化
- **来源**：[FB-07]，关联 [REQ-C3]
- **解决的问题**：修仙文化门槛对老人用户不友好；难度滑块缺乏参考
- **设计方向**：
  - 每个境界名称增加括号内通俗解释（如"练气一阶（初入修仙）"）
  - 愿望难度滑块1-10增加参考锚点（1=一杯奶茶，5=一次聚餐，10=全家旅行）

#### V2-F10：成就/里程碑系统
- **来源**：策划补充
- **解决的问题**：缺乏阶段性成就感，影响长期留存
- **设计方向**：
  - 首次Boss战胜利、连续打卡7天、某属性首次突破等节点增加庆祝反馈
  - 成就记录可在个人页查看
  - 家庭成员达成成就时推送家庭通知
