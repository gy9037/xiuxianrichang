# UI 审查报告 - V2.6 全页面（2026-04-13）

审查环境：375×667 视口（iPhone SE），通过 accessibility snapshot + 源码分析完成。

## 首页（home.js）

### P1 - 首屏信息密度低，关键信息被挤到折叠区
- 用户名 + 状态 badge 占一整行 page-header（约 44px），只有两个元素
- 境界卡片只有一行文字，独占一个 card（padding 32px + 内容 ≈ 60px）
- 修炼状态卡片同理，一行信息占一个 card
- 结果：属性雷达图被推到 y≈200px 以下，趋势图和推荐需要滚动

### P2 - 成就列表占据大量空间，信息价值低
- 6 个成就全部展开（含未解锁灰色项），每项约 40px，共 240px+
- 新用户全部未解锁，6 个灰色条目视觉沉闷
- 没有折叠/收起机制

### P3 - 退出登录放在页面最底部
- 需要滚到底才能看到，只是 12px 文字链接
- 不像常规设置入口

### P4 - 卡片间距统一 12px，缺乏层次
- 所有 card margin-bottom 都是 12px
- 境界、修炼状态、属性、趋势、推荐、成就之间没有视觉分组

## 家庭页（family.js）

### P5 - 成员列表严重膨胀（最严重的问题）
- 35 个成员全部展开，每个成员一行（头像 36px + padding 10px 上下），总高度约 1960px
- 大量测试账号混在真实用户中
- 没有折叠、分页或"显示更多"机制
- 每个成员：头像 + 名字 + 境界 badge + "属性总和 X.X"，一行一条，非常浪费空间

### P6 - 成员列表缺少环境状态显示
- 需求要求显示"头像+名字+境界+环境状态"
- 当前只有头像+名字+境界+属性总和，没有居家/生病/出差状态

### P7 - 动态 Feed 每条占用空间过大
- 每条：头像(36px) + 用户名 + 行为文本 + 时间 + 4 个表情按钮（min-height:44px，纵向 emoji+文字）
- 表情按钮组 padding:8px 14px + flex-wrap，4 个按钮一行约 60px 高
- 一条动态总高约 140-160px，20 条动态 = 3000px
- 没有分页或"加载更多"

### P8 - 团队愿望进度 35 人状态全部展开
- "出去玩"愿望的 teamProgress 显示全部 35 人"未挑战"，一行文本极长

## 行为页（behavior.js）

### P9 - 上报 tab 首屏内容偏少
- 新用户没有快捷入口时，首屏只有：tab bar + 标题 + 5 个分类按钮 + 折叠的"最近记录"
- 两步操作（选分类 → 选行为）才能开始输入

### P10 - tab bar 样式不够明显
- "上报"和"历史"两个 btn-small 紧贴，border-radius 拼接，视觉上像两个独立按钮而非 tab

## 愿望页（wish.js）

### P11 - 筛选区域占据首屏大量空间
- 筛选卡片：标题 + 类型 3 按钮 + 状态 5 按钮
- 375px 下状态按钮可能换行，整个筛选区约 150-180px
- 只有 4 个愿望时筛选 ROI 很低

### P12 - 团队愿望成员进度展示
- 默认前 5 人 + "查看全部"折叠，处理尚可
- 展开后 35 人状态文本极长

## 背包页（inventory.js）

### P13 - 整体尚可
- 空状态有引导按钮，体验可以
- 首次弹出炼化规则弹窗，信息量适中

## 全局问题

### P14 - 页面间 header 规范不统一
- 首页：flex（用户名+状态）
- 家庭页：纯文字"家庭"
- 愿望页：flex（标题+许愿按钮）
- 背包页：flex（标题+规则链接）
- 行为页：无 page-header，直接是 tab bar

### P15 - 所有页面 padding 统一 16px，未针对内容密度调整
- `.page { padding: 16px }` 全局统一
- 信息密集页面（家庭动态）和信息稀疏页面（空背包）用同样的间距

## 优化方案

### 方案一：首页布局重构

目标：首屏（667px 可视区 - 底部导航 70px = 597px）内展示核心信息。

1. 合并用户名+境界+修炼状态为一个紧凑的 hero 区域
   - 第一行：用户名（左）+ 环境状态 badge（右）
   - 第二行：境界 badge + 进度文字 + 修炼状态标签，一行搞定
   - 省掉两个独立 card 的 padding 开销，节省约 80px 高度

2. 属性雷达图保持不变（已经比较紧凑，max-width:200px）

3. 趋势图保持不变

4. 今日推荐精简为一行提示条（非独立 card）
   - 有推荐时：一行文字 + 右箭头，点击跳转
   - 无推荐时：不显示

5. 成就区域改为折叠式
   - 默认只显示"成就 X/6"摘要行
   - 点击展开完整列表
   - 或者只显示最近解锁的 1-2 个

6. 退出登录移到某个设置入口（比如点击用户名弹出菜单）

预估效果：首屏可以完整展示 hero + 雷达图 + 趋势图，核心信息一屏可见。

### 方案二：家庭页成员列表紧凑化

1. 成员列表改为网格/紧凑排版
   - 每个成员：圆形头像(28px) + 名字(12px) + 境界小标签 + 环境状态小圆点
   - 横向排列，每行 3-4 个，类似通讯录头像墙
   - 或者改为横向滚动的 avatar 条

2. 默认最多展示 5 个成员
   - 超过 5 个显示"+N"按钮，点击展开全部
   - 排序：按最近活跃时间降序，活跃用户优先

3. 添加环境状态显示
   - 头像右下角小圆点：绿色=居家，红色=生病，蓝色=出差

### 方案三：家庭动态 Feed 优化

1. 表情按钮组改为横向紧凑排列
   - 去掉纵向的文字标签（赞/强/悟/定），只保留 emoji + 计数
   - 按钮尺寸从 min-height:44px 缩小到 32px（仍满足触控要求，因为横向间距足够）
   - 4 个按钮一行，总高度从 60px 降到 32px

2. Feed 添加分页
   - 默认显示最近 10 条
   - 底部"加载更多"按钮

3. 动态文本精简
   - "完成了 读书 （凡品） → 获得 般若灵茶" 可以更紧凑
   - 考虑把品质和道具名放在同一行

### 方案四：行为页微调

1. tab bar 改为更明显的 segment control 样式
   - 统一圆角，选中态用背景色区分

2. 考虑在无快捷入口时，默认展开第一个分类
   - 减少一步操作

### 方案五：愿望页筛选优化

1. 筛选区域默认折叠
   - 只显示"筛选"标题 + 当前筛选条件摘要
   - 点击展开完整筛选面板
   - 愿望数量少于 5 个时隐藏筛选
