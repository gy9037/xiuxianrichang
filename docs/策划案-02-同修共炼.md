## 一、背景与设计目标

### 核心定位

任务系统是行为挑战系统，与愿望系统互补：

| 系统 | 定位 | 示例 |
|------|------|------|
| 愿望系统 | 奖励系统（想要什么） | 买摩托车、买奶茶、出去玩 |
| 任务系统 | 行为挑战系统（要做什么） | 读一本书、减肥2KG、跑半马 |

愿望系统通过 Boss 战做属性验证，任务系统通过实际行为完成做验证。

### 设计原则

- 投票民主：任务需要家庭成员投票通过才能生效
- 失败无惩罚：避免家庭场景中的负面情绪
- 灵石驱动：奖励和质押统一使用灵石体系

## 二、三种任务来源

### 2.1 系统悬赏（原秘境探索）

系统随机发布的每日轻量任务，引导用户用新视角观察生活。

任务池共 85 条，分五类：

| 类别 | 数量 | 示例 |
|------|------|------|
| 发现类 | 20 | 拍一张今天看到的最美的光影 |
| 行动类 | 18 | 用左手完成一件平时用右手做的事 |
| 社交类 | 15 | 给一位家人写一张纸条 |
| 感知类 | 17 | 闭眼一分钟，记录听到的所有声音 |
| 思考类 | 15 | 写下今天学到的一件新事 |

发布规则：
- 每日自动发布 1 条系统悬赏（从任务池随机抽取，保证近期不重复）
- 奖励：完成后获得灵石（数值待定）
- 支持照片提交作为完成证据

### 2.2 自我悬赏

用户给自己发起的挑战目标。

- 发起者设定：类型（五属性之一）、目标描述、截止时间、自动结算条件（可选）
- 无需投票，直接生效
- 完成后由系统自动判定或自行标记完成
- 奖励由系统产出灵石

### 2.3 他人悬赏 / 系统任务（挑战型）

由家庭成员发起的挑战任务，需要投票通过。

**发起流程：**
1. 任何成员发起挑战任务，设定：类型（五属性之一）、目标、截止时间、奖励（灵石+对应类型道具）
2. 全服公告，所有成员收到通知
3. 成员投票赞成/反对，超过 51% 赞成票任务才能生成（发起者算赞成票，2人家庭需全票）
4. 投票赞成时可勾选"一起挑战"（同修共炼标记），勾选者成为挑战者
5. 未勾选的成员为旁观者

**完成与判定流程：**
6. 截止日弹出提交框，挑战者提交完成信息（文字+可选照片）
7. 其他成员判定是否完成（51%原则，挑战者本人不算票）
8. 奖励由系统凭空产出

**悬赏模式：**
- 发起者出资灵石作为奖励
- 其他成员接取完成后获得悬赏
- 适合家庭场景（如家长悬赏让孩子完成特定任务）

## 三、自动结算目标类型

从策划案-04（家庭挑战赛）吸收的三种可自动判定的目标类型：

| 目标类型 | 判定方式 | 示例 |
|---------|---------|------|
| 行为次数 | 统计周期内指定类别的行为提交次数 | 本周运动 5 次 |
| 连续天数 | 统计周期内指定类别的连续打卡天数 | 连续早起 7 天 |
| 属性积累 | 统计周期内指定属性的临时值积累量 | 本月悟性积累 20 点 |

自动结算的任务不需要人工判定，系统在截止时间自动计算结果。

---

## 四、竞争与合作模式

| 模式 | 说明 | 奖励分配 |
|------|------|---------|
| 合作型 | 所有挑战者共同达成目标 | 全员达成则全员获奖 |
| 竞争型 | 挑战者之间比拼 | 排名靠前者获得更多奖励 |

---

## 五、图片存储原则

从策划案-03（秘境探索）吸收：

- 保存原图，不压缩不删除，图片是生活记录的核心资产
- 可做无损/接近无损处理（去除冗余EXIF等），不降分辨率
- 存储压力通过导出或分层存储解决，不牺牲质量
- 存储到 R2，家庭成员可互相查看
- 探索图鉴保留，作为照片记录的展示层，可用于年度总结回顾（**待补充**：图鉴页面设计、分类逻辑、年度总结呈现方式尚未细化，需单独补充策划后再出技术方案）

## 六、数据库设计

### 新增表：`quests`

```sql
CREATE TABLE quests (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  family_id INTEGER NOT NULL,
  creator_id INTEGER NOT NULL,
  type TEXT NOT NULL,              -- 'system'|'self'|'bounty'|'challenge'
  title TEXT NOT NULL,
  description TEXT,
  category TEXT,                   -- 五属性之一（可选）
  goal_type TEXT,                  -- 'manual'|'behavior_count'|'streak_days'|'attr_accumulate'
  goal_config TEXT,                -- JSON: {target: 5, category: '学习', period: '2026-04-W17'}
  mode TEXT DEFAULT 'cooperative', -- 'cooperative'|'competitive'
  reward_stones INTEGER DEFAULT 0,
  reward_items TEXT,               -- JSON: 奖励道具配置
  status TEXT DEFAULT 'voting',    -- 'voting'|'active'|'judging'|'completed'|'failed'|'cancelled'
  vote_deadline DATETIME,
  deadline DATETIME,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  completed_at DATETIME
);
```

### 新增表：`quest_participants`

```sql
CREATE TABLE quest_participants (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  quest_id INTEGER NOT NULL,
  user_id INTEGER NOT NULL,
  role TEXT NOT NULL,              -- 'challenger'|'observer'|'bounty_taker'
  vote TEXT,                       -- 'approve'|'reject'|null
  progress TEXT,                   -- JSON: 自动结算进度
  submission TEXT,                 -- JSON: {text, photo_urls}
  result TEXT,                     -- 'completed'|'failed'|null
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(quest_id, user_id),
  FOREIGN KEY (quest_id) REFERENCES quests(id)
);
```

### 新增表：`quest_judgments`

```sql
CREATE TABLE quest_judgments (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  quest_id INTEGER NOT NULL,
  target_user_id INTEGER NOT NULL,
  judge_user_id INTEGER NOT NULL,
  verdict TEXT NOT NULL,           -- 'pass'|'fail'
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(quest_id, target_user_id, judge_user_id),
  FOREIGN KEY (quest_id) REFERENCES quests(id)
);
```

### 新增表：`system_quest_pool`

```sql
CREATE TABLE system_quest_pool (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  category TEXT NOT NULL,          -- 'discover'|'action'|'social'|'perception'|'thinking'
  title TEXT NOT NULL,
  description TEXT NOT NULL,
  requires_photo INTEGER DEFAULT 0
);
```

## 七、API 设计

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/quests | 任务列表（支持 status 筛选） |
| POST | /api/quests | 创建任务 |
| POST | /api/quests/:id/vote | 投票（approve/reject + 是否参与） |
| POST | /api/quests/:id/submit | 提交完成信息 |
| POST | /api/quests/:id/judge | 判定某参与者是否完成 |
| GET | /api/quests/:id | 任务详情（含参与者、进度、判定状态） |
| GET | /api/quests/daily | 获取今日系统悬赏 |
